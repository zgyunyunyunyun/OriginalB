# 微信小游戏导出流程（当前项目可执行版）

> 目标：基于当前项目 `OriginalB`，优先导出并验证微信小游戏包。

## 0. 当前适配状态（已确认）

- 已接入微信导出工具链目录：`Assets/WX-WASM-SDK-V2/`
- 平台服务分发骨架已就绪：`Assets/Scripts/Platform/Bootstrap/PlatformBootstrap.cs`
- 默认 WebGL 宏已切到微信：`PLATFORM_WECHAT`
- 平台服务自动启用条件已修正：存在 `PLATFORM_WECHAT` 或 `PLATFORM_DOUYIN` 时会启用对应平台服务

## 1. Unity 内构建前检查

1. 打开场景：`Assets/Scenes/MainScene.unity`
2. 确认 Build Settings 目标平台为 `WebGL`
3. 打开 `Edit > Project Settings > Player > Other Settings > Scripting Define Symbols`
4. 确认 WebGL 宏包含：
   - `PLATFORM_WECHAT`
5. 确认不包含：
   - `PLATFORM_DOUYIN`

> 如后续要切回抖音，只需替换宏为 `PLATFORM_DOUYIN`。

## 2. 导出微信小游戏工程

1. 在 Unity 菜单中打开微信导出面板（由 `WX-WASM-SDK-V2` 提供）
2. 常用建议项：
   - 导出目录：`OutGame/OriginalB-WX`（建议与抖音包分目录）
   - 代码压缩：开启（发布包建议）
   - 开发调试：联调阶段开启，提审前关闭不必要调试输出
3. 点击导出，等待导出完成

## 3. 微信开发者工具导入与运行

1. 打开微信开发者工具
2. 导入第 2 步生成的小游戏工程目录
3. 配置 `AppID`（你的小程序/小游戏真实 AppID）
4. 编译并运行，重点查看控制台：
   - 必须出现平台诊断日志
   - 必须包含：`PlatformContext=WeChatPlatformContext`
   - 建议核对：`InputService/StorageService/AuthService/AdService/ShareService/AnalyticsService` 均为 `WeChat*`

## 4. 最小回归（上线前）

按文档执行：

- `Docs/小游戏真机前检查清单.md`
- `Docs/小游戏最小回归用例表.md`

最小必测链路：

1. 启动进入主界面
2. 拖拽/点击交互正常（触控）
3. 通关与失败逻辑正常
4. 关闭重进后关键存档保留
5. 弱网/断网下不崩溃

## 5. 常见问题定位

### 5.1 包跑成了 Common 或 Douyin

- 原因：宏配置错误或未生效
- 处理：
  1. 再次确认 WebGL 宏仅保留 `PLATFORM_WECHAT`
  2. 重新导出微信工程
  3. 在日志中确认 `PlatformContext=WeChatPlatformContext`

### 5.2 导出后微信工具编译失败

- 先确认 Unity 版本与插件版本匹配（当前工程 Unity 为 `2022.3.62f2c1`）
- 清空导出目录后重新导出
- 若涉及 AppID 或权限项报错，先在微信开发者工具项目配置内修正

### 5.3 真机体验与编辑器不一致

- 优先以真机结果为准
- 按 `Docs/小游戏MVP体验报告模板.md` 记录差异并回归

## 6. 提审前建议

1. 关闭无关调试日志
2. 完成真机最小回归
3. 产出一份体验报告：`Docs/小游戏MVP体验报告模板.md`
4. 再执行一次完整导出，锁定提审包
